WSL + Docker Compose 一键部署 GeoServer + PostGIS

WSL + Docker Compose 一键部署 GeoServer + PostGIS 完整教程

在 Windows 环境下做 GIS 开发，传统本地安装 GeoServer + PostGIS 步骤繁琐、环境冲突多、迁移麻烦。本文使用 WSL2 + Docker + Docker Compose，提供一套可直接复制使用的配置文件与脚本，实现一键启动、数据持久化、环境隔离，开发/测试/演示都非常稳定。

一、环境说明

  • 操作系统：Windows 10/11（开启 WSL2）
  • 容器工具：Docker Desktop（开启 WSL 集成）
  • 镜像：
    • kartoza/postgis:16-3.4（PostgreSQL 16 + PostGIS 3.4）
    • kartoza/geoserver:2.25.0
  • 功能：
    • PostGIS 空间数据库自动初始化
    • GeoServer 后台运行、数据持久化
    • 服务依赖健康检查（先启动数据库）
    • 一键启停/重启/查看日志

七、WSL 注意事项

  1. 必须使用 WSL2，WSL1 不支持 Docker 正常运行
  2. 数据卷默认使用 Docker 卷，性能稳定、不建议挂载到 /mnt/c
  3. Windows 直接访问 localhost 即可，无需手动转发端口
  4. 若 8080/5432 被占用，修改 ports 映射（如 8081:8080）
  5. 生产环境务必修改所有默认密码

八、总结

  • 简单：一条命令启动整套 GIS 环境
  • 干净：Docker 隔离，不污染系统环境
  • 稳定：健康检查 + 依赖顺序，启动无报错
  • 可迁移：复制两个文件即可在任意 Linux/WSL 机器重建环境
